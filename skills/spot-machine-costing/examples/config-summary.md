# 配置摘要输出示例

下面是 skill 期望从服务器图片 / 截图 / 文字描述中提取的标准输出格式。

## 模板

```text
CPU：{型号} *{数量}
GPU：{型号} *{数量}
内存：{容量} {类型} {频率} *{数量}
系统盘：{容量} {接口} *{数量}
数据盘：{容量} {接口} *{数量}
```

## 真实例子

### 超微 B300 / 数据盘用 E1.S 接口

```text
CPU：Intel Xeon 6767P *2
GPU：NVIDIA HGX B300 NVL8 8-GPU *1
内存：128G DDR5 6400 *32
系统盘：1.92T NVMe M.2 *1
数据盘：7.68T NVMe E1.S *4
```

匹配价库（v0.1.3）：

- 数据盘接口 = `E1.S`，品牌 = 「超微」 → 命中价库 `7.68T NVMe E1.S` 接口下的「超微」品牌专属行。
- 价格按该行当前的更换单价 / 售卖单价读取，**不在 skill 文档里写死**。

### 广达 B300 / 数据盘用 E1.S 接口

```text
CPU：Intel Xeon 6767P *2
GPU：NVIDIA HGX B300 8-GPU *1
内存：128G DDR5 6400 *32
系统盘：1.92T NVMe M.2 *1
数据盘：7.68T NVMe E1.S *4
```

匹配价库（v0.1.3）：

- 数据盘接口 = `E1.S`，品牌 = 「广达」 → 命中 `E1.S` 接口下的「广达」品牌专属行。
- 价格按该行当前的金额读取。

### 华硕 B300 / 数据盘用 U.2 接口

```text
CPU：Intel Xeon 6767P *2
GPU：NVIDIA HGX B300 8-GPU *1
内存：96G DDR5 6400 *32
系统盘：960G NVMe M.2 *1
数据盘：3.84T NVMe U.2 *8
```

匹配价库（v0.1.3）：

- 数据盘接口 = `U.2`，品牌 = 「华硕」 → 命中 `U.2` 接口下的「华硕」品牌专属行；找不到专属行时回落到 `U.2` 通用行。
- 价格按命中的那一行读取。

## 反例（不要这样做）

### ❌ 不从容量反推接口

```text
# 错：看到 3.84T 就默认写 E1.S
数据盘：3.84T E1.S *4

# 对：只写能从原文确认的信息；接口未明时写 待确认
数据盘：3.84T 接口待确认 *4
```

### ❌ 把物理尺寸识别成数量

```text
# 错：22x110mm 是 M.2 物理尺寸，不是数量
系统盘：1.92T NVMe M.2 *22x110mm *2

# 对：保留原始尺寸作为备注，数量按 *2 明确表达
系统盘：1.92T NVMe M.2 (22x110mm) *2
```

### ❌ 数量空缺仍要凑整

```text
# 错：把 4 条 × 64GB 强写成 256GB *1
内存：256G DDR5 6400 *1

# 对：保留 *4 表达
内存：64G DDR5 6400 *4
```

### ❌ 在示例 / 反例里硬编码价格

```text
# 错：示例里直接写「更换价 X 万元/块」「售卖价 Y 万元/块」
数据盘接口 = E1.S → 强制更换价 X 万元/块

# 对：示例只描述匹配逻辑，价格一律说明「按当前价库读取」
数据盘接口 = E1.S → 命中 E1.S 接口下的品牌专属行，按该行当前价格读取
```

## 待确认字段

五类中任何一类无法从输入确认，必须写 `待确认`，不能靠常识猜：

```text
CPU：待确认
GPU：NVIDIA HGX B300 8-GPU *1
内存：待确认
系统盘：1.92T NVMe M.2 *1
数据盘：待确认
```

skill 在汇报结果时应明确告诉用户哪几项是 `待确认`。
