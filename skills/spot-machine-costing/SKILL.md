# 现货机器成本核算

## 执行合同

调用本 skill 后，只做一件事：把本次机器作为新记录写入现货机器成本核算工作簿，并完成配置识别、标准成本拆分、公式补齐和自检。

## 绝对规则

- 只操作主表 `Summary`；价格只读 `标准单价database`。
- 所有列都按表头语义定位，禁止写死列字母或单元格。
- 所有金额必须从当前 `标准单价database` 读取，禁止用记忆、历史快照、聊天内容或外部网页价格。
- 默认在 `Summary` 最后一条有效数据后新增一行；只有用户明确指定行，才写指定行。
- 覆写已有数量、单价、公式值或业务数据前，必须先告知用户并等确认。
- 表头、价库、公式来源无法定位时，停止对应写入并说明缺什么。
- 不确定的配置、价格、字段或匹配结果写 `待确认`，不要猜。

## 最少输入

- 必须有：服务器图片/配置文本、品牌、数量。
- 可选：交易方、采购价格、实际付款总价、备注。
- 缺品牌且影响价格，或缺数量时，先问用户，不继续写。

## 必跑流程

### 1. 建字段映射

读取 `Summary` 和 `标准单价database` 的表头，建立字段名到列位置的映射。

`Summary` 至少定位：Item/序号、品牌、数量、配置图/来源、配置识别结果、成本拆分字段、原机器单价、实际付款总价、等效标准配置单价、核算后总价、备注。

`标准单价database` 至少定位：产品类型、标准规格、匹配关键词、适用品牌/条件、更换单价、拆机售卖单价、单位、备注。

字段名可轻微不同；无法确认含义时停止，不猜列。

### 2. 新增基础行

- 继承上一条有效数据行的样式和公式结构。
- 写入用户给的品牌、数量、配置来源、采购价格、交易方、备注等。
- Item/序号按现有连续规则延续。
- 未提供且非关键的信息按原表习惯留空或写 `待确认`。

格式校正：Item/序号列居中加粗；其它本次写入字段左对齐、正常字重；图片列按原图片列样式处理。写后若发现非 Item 列被误设为居中或加粗，立即修正。

### 3. 写配置摘要

配置摘要必须固定输出八类，八类都要出现：

```text
CPU：...
GPU：...
内存：...
系统盘：...
数据盘：...
管理网卡：...
存储网卡：...
计算网卡：...
```

识别规则：缺失写 `待确认`；只认明确数量表达（如 `*2`、`×2`、`x2`、`2块`、`2条`）；不要把 `22x110mm` 等物理尺寸识别成数量；不要只凭容量反推接口（如 `3.84T` 不能反推 `E1.S` 或 `U.2`）。

### 4. 从价库取价

每次都按当前 `标准单价database` 读取价格，不缓存历史值。

匹配失败、品牌条件不符、价格为空时，对应价格或拆分项写 `待确认`。

### 5. 做标准成本拆分

按当前机器配置与标准配置逐项比对，写入拆分成本字段。

- 比对粒度：`部件类型 + 规格 + 数量`。
- 同规格数量不同：只计算数量差。
- 规格不同：标准规格按更换价计入，原规格按拆机售卖价抵扣。
- 禁止按总容量差额粗算。
- 新品牌、新规格、新接口或规则不明：写 `待确认`。

价库匹配顺序：先按接口类型（如 `E1.S` / `U.2` / `M.2`）命中规格行，再叠加品牌条件；有品牌专属行用专属行，没有则用该接口通用行。品牌缺失且影响价格时先问用户。

### 6. 补齐公式

对公式字段，从上一行同字段复制公式；上一行无效时，向上找最近有效公式行；复制时调整到目标行引用。找不到公式来源时停止，不手写新公式。

### 7. 最终自检

写入后必须回读检查：

- 基础信息是否写入正确列。
- 配置摘要是否包含 CPU/GPU/内存/系统盘/数据盘/管理网卡/存储网卡/计算网卡八类。
- 数量是否没有误读物理尺寸。
- 价格是否来自当前 `标准单价database`。
- 拆分字段是否按接口优先、品牌条件匹配。
- 公式是否复制成功且引用目标行。
- 非 Item 列是否保持左对齐、正常字重。
- 所有 `待确认` 是否在回复中逐项说明。

## 回归测试

只有修改本 skill、价库结构或拆分逻辑时，才使用测试参考表做回归；日常新增机器不要求用户验证，由执行者完成自检。
