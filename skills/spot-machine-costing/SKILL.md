# 现货机器成本核算

## 核心原则

- 用户明确调用本 skill 后，按用户提供的信息主填指定的现货机器成本核算表。
- 所有列都通过表头语义动态定位，不绑定固定列字母。
- 只填写本次机器核算需要的字段，不改表格格式。
- 不确定的信息写 `待确认`，不猜配置、不猜价格、不猜字段位置。
- 表头、价库、公式来源找不到时，停止写入并说明缺什么。

## 用户输入

用户通常会提供：

- 服务器图片或配置截图
- 品牌
- 数量
- 可选：交易方、采购价格、实际付款总价、备注等

如果缺少影响核算的关键信息，例如品牌或数量，先问用户补齐。

## 工作簿对象

- 主表：`Summary`
- 标准价库：`标准单价database`
- 标准配置：以主表中的表头或现有标准配置标记定位，不写死单元格
- 测试参考表：仅用于规则变更后的回归检查

## 执行流程

### 1. 读取表结构

1. 打开用户指定或当前上下文中的现货机器成本核算工作簿。
2. 读取主表的表头行，建立“字段名 → 列位置”映射。
3. 读取标准价库的表头行，建立价库字段映射。
4. 根据表头语义定位这些字段：
   - item / 序号 / 编号
   - 品牌
   - 数量
   - 配置图片 / 原始配置 / 图片来源
   - 配置识别结果 / 配置摘要
   - 标准化拆分成本相关字段
   - 原机器单价
   - 实际付款总价
   - 等效标准配置单价
   - 核算后总价
   - 备注

字段名可有轻微差异，优先按现有表头含义匹配；无法确认时停止，不猜列。

### 2. 写入基础信息

1. 找到需要新增或填写的目标行。
2. 根据动态表头写入用户提供的品牌、数量、图片/配置来源、采购价格、交易方、备注等。
3. item / 序号字段按现有行的连续规则延续。
4. 用户未提供的信息保持空白或写 `待确认`，按该字段现有习惯处理。

### 3. 识别配置摘要

从用户提供的图片、截图或配置文本中识别配置，写入“配置识别结果 / 配置摘要”字段。

固定输出五类：

```text
CPU：...
GPU：...
内存：...
系统盘：...
数据盘：...
```

规则：

- 五类都必须出现。
- 缺失字段写 `待确认`。
- 不从容量反推接口，例如不能只凭 `3.84T` 判断 `E1.S` 或 `U.2`。
- 忽略 `22x110mm` 这类物理尺寸，不能识别成数量。
- 数量只认明确表达，例如 `*2`、`×2`、`x2`、`2块`、`2条` 等。
- 输出格式要便于和标准配置逐项对比。

### 4. 读取标准价库

只从标准价库获取价格和匹配规则。

价库字段通过表头动态定位，至少包括：

- 产品类型
- 标准规格
- 匹配关键词
- 适用品牌/条件
- 更换单价
- 拆机售卖单价
- 单位
- 备注

规则：

- 价格单位沿用表内现有口径。
- 找不到规格、品牌条件不匹配、价格为空时，对应核算项写 `待确认`。
- 表内价库是唯一价格来源；不要把聊天记忆里的价格当作最终来源。

### 5. 成本拆分

用当前机器配置和标准配置对比，写入标准化拆分成本字段。

对比逻辑：

- 按 `部件类型 + 规格 + 数量` 比对。
- 同规格数量不同：只计算数量差。
- 规格不同：标准规格按更换价计入，原规格按拆机售卖价抵扣。
- 不按总容量差额粗算。
- 没有明确规则的新品牌、新规格、新接口，写 `待确认`。

品牌条件：

- 价格规则必须结合品牌判断。
- 如果品牌影响价格但用户未提供品牌，先问用户。
- 如果价库里存在品牌专属规则，优先使用品牌专属规则；否则使用通用规则。

### 6. 公式补齐

对需要公式的核算字段：

1. 根据表头定位需要公式的字段。
2. 从上一行复制同字段公式。
3. 上一行无效时，向上找最近有效公式行。
4. 复制公式时按目标行调整引用。
5. 找不到公式来源时停止，不手写新公式。

### 7. 自动核对

写入后自行检查：

- 基础信息是否写入正确字段。
- 配置摘要是否包含五类。
- 数量是否没有把物理尺寸误识别成数量。
- 价库是否命中。
- 拆分字段是否按表头语义写入。
- 公式字段是否成功复制并引用目标行。
- 有 `待确认` 时，明确告诉用户是哪一项缺信息。

## 规则变更回归

只有当本 skill 的规则、价库结构、拆分逻辑发生变化时，才使用测试参考表做回归检查。

日常新增机器不要求用户验证；由执行者完成自动核对。遇到新规则无法判断时，再向用户确认。

## 失败处理

- 图片看不清：对应配置写 `待确认`。
- 品牌缺失且影响价格：先问品牌。
- 数量缺失：先问数量。
- 价库无匹配：对应价格/拆分项写 `待确认`。
- 表头无法匹配：停止写入，避免错列。
- 公式来源缺失：停止写入公式，等待用户确认。
