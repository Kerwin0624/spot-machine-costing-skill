# Spot Machine Costing Skill

> 版本：**v0.1.4**（2026-06-26）  
> 变更历史：[CHANGELOG.md](./CHANGELOG.md)

一个面向 Mavis / mavis agent 的「现货机器成本核算」skill 模板：用户贴服务器图片 / 配置截图 + 品牌 + 数量 + 交易信息，agent 按电子表格（飞书 Sheet）的现有表头写入新机器，识别配置摘要、从标准价库读取价格、按规则拆分成本差异，并补齐核算公式。

## 适用场景

使用这个 skill 处理：

- 现货服务器配置识别（从图片 / 截图 / 文字）
- 按标准配置核算机器成本差异
- 标准价库驱动的内存、硬盘、GPU 等部件「更换价 / 拆机售卖价」拆分
- 在电子表格主表底部新增机器行，自动延续 item 计数并补齐公式

不要用它处理：

- 普通电子表格编辑（无关成本核算）
- 没有标准价库支持的自由估价
- 与服务器成本核算无关的报价

## 核心特点

- **动态表头定位**：通过表头语义定位列，不绑定固定列字母。
- **价库是唯一价格来源**：所有金额从 `标准单价database` 读取；skill / 文档里不出现任何具体金额。
- **保守填表**：配置 / 价格 / 字段位置不确定时写 `待确认`，不靠猜。
- **公式复用**：从上一行或最近有效行复制公式，不手写业务公式。
- **新增行格式统一（v0.1.4+）**：仅 `Item` / 序号 / 编号列居中加粗，其余写入列左对齐、正常字重。
- **接口匹配先行（v0.1.3+）**：数据盘按接口（`E1.S` / `U.2` / `M.2`）优先匹配价库，再叠加品牌条件。
- **品牌是匹配条件，不是覆盖项**：同一接口不同品牌，命中价库对应的品牌专属行；找不到品牌专属行时回落到该接口的通用行。
- **覆写需先告知**：数量 / 单价 / 公式值的覆写必须先告知用户并等待确认。
- **自动核对**：写入后回读检查配置摘要、价库命中、拆分字段和公式引用。

## v0.1.4 变更速览

### 新增行格式统一

- `Item` / 序号 / 编号列：居中对齐、加粗。
- 除 `Item` / 序号 / 编号列以外，本次写入的所有输入字段、拆分字段、公式字段、备注字段：左对齐、正常字重。
- 配置图字段按原表图片列样式处理；不要影响同一行其它字段的对齐和字重。

## v0.1.3 变更速览

### 1. 接口匹配优先 + 品牌作为匹配条件（不是覆盖）

拆分匹配优先级：

1. 接口匹配（`E1.S` / `U.2` / `M.2` ...）
2. 品牌匹配（在已命中的接口行里继续匹配）
3. 通用 fallback

例：

- 数据盘 `7.68T NVMe E1.S`：先命中价库里 `E1.S` 接口的所有行；品牌是「超微」则命中「`7.68T NVMe E1.S` / 超微」行；品牌是「广达」则命中「`7.68T NVMe E1.S` / 广达」行。
- 数据盘 `7.68T NVMe U.2`：命中 `U.2` 接口的所有行；品牌是「华硕」等则按该接口的通用行计入。

**为什么不同接口价格有差异**：超微 / 广达用的数据盘是 `E1.S` 接口，华硕等用的是 `U.2` 接口——**不是**「`E1.S` 覆盖品牌」，而是「因为这些品牌用了 `E1.S` 接口，价格才相对 `U.2` 更高」。使用 `U.2` 接口的品牌不在 `E1.S` 规则的覆盖范围内。

### 2. 价格不硬编码

- skill / 文档不再写任何具体金额。
- 价格每次按 `标准单价database` 实时读取，价库变了 skill 行为自然跟着变。
- 任何硬编码价格（无论是聊天记忆、文档、外部网页还是历史快照）一律视为错误信息源，回退到价库读取。

### 3. 覆写需先告知

数量 / 单价 / 公式值的覆写必须先告知用户并等待确认。

## 推荐表格结构

skill 不依赖固定列号，但推荐工作簿包含以下对象。

### 主表 `Summary`

存放单台机器的行级数据。常见字段（顺序可调）：

- `Item` / `序号` / `编号`
- `品牌` / `数量`
- `原机器单价` / `原机器总价-实际付款`
- `等效2T配置单价` / `核算后总价`
- `更换内存规格` / `数量` / `单价`
- `更换系统盘规格` / `数量` / `单价`
- `更换数据盘规格` / `数量` / `单价`
- `更换网卡规格-25G` / `数量-25G` / `单价-25G`
- `更换网卡规格-200G` / `数量-200G` / `单价`
- `售卖内存规格` / `数量` / `单价`
- `售卖系统盘规格` / `数量` / `单价`
- `售卖数据盘规格` / `数量` / `单价`
- `售卖网卡规格` / `数量` / `单价`

公式示例（按目标行调整引用）：

- `等效2T配置单价`：  
  `=D{r} + Z{r}*AA{r} + AC{r}*AD{r} + AF{r}*AG{r} + AI{r}*AJ{r} + AL{r}*AM{r} - AO{r}*AP{r} - AR{r}*AS{r} - AU{r}*AV{r} - AX{r}*AY{r}`
- `核算后总价`：  
  `=C{r}*F{r}`
- `原机器总价-实际付款`：  
  `=D{r}*C{r}`

### 标准价库 `标准单价database`（唯一价格来源）

至少包含字段：

- `产品类型` / `标准规格` / `匹配关键词` / `适用品牌/条件`
- `更换单价-全新` / `售卖单价-拆机` / `单位` / `备注`

单位沿用 Summary 的 Y:AV 口径（**万元**），不做元/万元换算。

> **价库如何配置接口 × 品牌交叉**：建议同一接口下按品牌拆行，例如：
>
> - `7.68T NVMe E1.S` / 适用品牌「超微」 → 一行
> - `7.68T NVMe E1.S` / 适用品牌「广达」 → 一行
> - `7.68T NVMe U.2` / 适用品牌「通用」 → 一行
>
> skill 命中「`7.68T NVMe E1.S`」时，再按品牌挑选对应行；找不到品牌专属行时回落到该接口的通用行。

## 调用示例

用户："帮我用这个 skill 算一下这台服务器的核算价。"  
附：服务器图片、品牌「超微」、数量 32、交易方「Vendor Inc」、付款总价若干万元。

agent 流程：

1. 解析工作簿 → 找到 `Summary` / `标准单价database`。
2. 读表头 → 建立字段 → 列字母映射。
3. 识别配置（5 类）：CPU / GPU / 内存 / 系统盘 / 数据盘。
4. 读价库 → 按接口优先匹配 → 再按品牌匹配 → 命中后用价库行的金额。
5. 计算拆分项 → 写入 `Summary` 主表最后一行。
6. 从上一行复制公式 → 补齐本行公式。
7. 回读核对 → 报 `待确认` 项。

```text
用户提供图片/截图/品牌/数量/交易
   │
   ▼

   │
   ▼
读表头 → 字段映射 → 列定位
   │
   ▼

   │
   ▼

   │
   ▼
按接口匹配价库行 → 按品牌命中专属行 → 取金额
   │
   ▼

   │
   ▼
从上一行复制公式 → 调整引用
   │
   ▼
回读核对 → 报告结果 / 待确认项
```

## 失败处理

| 现象 | 处理 |
|---|---|
| 图片 / 截图不清晰 | 对应配置写 `待确认` |
| 品牌缺失且影响价格 | 先问用户 |
| 数量缺失 | 先问用户 |
| 价库无匹配规格 / 接口 | 对应价格项写 `待确认` |
| 表头无法解析 | 停止写入，避免错列 |
| 公式来源缺失 | 停止写入公式，等待用户确认 |
| 覆写已有数值 | 先告知用户并等确认，再写 |
| 出现硬编码价格（聊天记忆、文档、外部网页、历史快照） | 一律视为错误信息源，回退到 `标准单价database` 读取 |
