# Spot Machine Costing Skill

一个用于服务器现货机器成本核算的 Mavis skill 模板。

它适用于这样的工作流：用户提供服务器图片、品牌、数量、采购或交易信息，agent 根据电子表格现有表头写入新机器信息，识别配置摘要，读取标准价库，拆分配置差异成本，并补齐核算公式。

## 适用场景

使用这个 skill 处理：

- 现货服务器配置识别
- 服务器图片 / 配置截图转结构化摘要
- 按标准配置核算机器成本差异
- 根据标准价库拆分内存、硬盘、GPU 等部件更换价和拆机回收价
- 在电子表格底部新增机器行、自动延续 item 计数并补齐公式

不要用它处理：

- 普通电子表格编辑
- 与服务器成本核算无关的报价
- 企业工商信息补全或供应商尽调
- 没有标准价库支持的自由估价

## 核心特点

- **动态表头定位**：通过表头语义定位列，不绑定固定列字母，后续增删列也更稳。
- **表内价库优先**：从 `标准单价database` 读取价格，不把价格硬编码到 skill。
- **保守填表**：配置、价格、字段位置不确定时写 `待确认`，不靠猜。
- **公式复用**：从上一行或最近有效行复制公式，不手写新的业务公式。
- **自动核对**：写入后检查配置摘要、价库命中、拆分字段和公式引用。
- **可回归验证**：规则变化时可以用测试参考表做回归检查。

## 推荐表格结构

这个 skill 不依赖固定列号，但推荐工作簿包含以下对象。

### 主表

默认主表名：`Summary`

推荐字段语义：

| 字段语义 | 示例表头 |
| --- | --- |
| 序号 | `item`、`序号`、`编号` |
| 品牌 | `品牌`、`厂商` |
| 数量 | `数量`、`台数` |
| 配置来源 | `配置图片`、`原始配置`、`图片来源` |
| 配置摘要 | `配置识别结果`、`配置摘要` |
| 原机器单价 | `原机器单价`、`采购单价` |
| 实际付款总价 | `实际付款总价`、`原机器总价-实际付款` |
| 等效标准配置单价 | `等效标准配置单价`、`等效2T配置单价` |
| 核算后总价 | `核算后总价` |
| 备注 | `备注` |

标准化拆分成本字段可以放在任意列，只要表头能表达部件类型、规格、数量、单价、金额等含义。

### 标准价库

默认价库表名：`标准单价database`

推荐字段：

| 字段 | 说明 |
| --- | --- |
| `产品类型` | 内存、系统盘、数据盘、GPU 等 |
| `标准规格` | 标准化规格名称 |
| `匹配关键词` | 用于匹配配置摘要中的关键词 |
| `适用品牌/条件` | 品牌专属规则或通用规则 |
| `更换单价` | 标准规格更换成本 |
| `拆机售卖单价` | 原规格拆机回收价 |
| `单位` | 台、块、条等 |
| `备注` | 特殊说明 |

价库是唯一价格来源。新品牌、新规格或新接口没有价格时，应先补价库，再继续核算。

### 测试参考表

可选表：`测试参考表`

用途是规则变更后的回归检查。日常新增机器不要求人工验证测试表。

## 安装

将 `skills/spot-machine-costing` 复制到你的 Mavis agent skills 目录。

```bash
cp -R skills/spot-machine-costing ~/.mavis/agents/main/skills/
```

如果你使用的是其他 agent 名称，把 `main` 替换成对应 agent 目录名。

```bash
cp -R skills/spot-machine-costing ~/.mavis/agents/<agent-name>/skills/
```

安装后，重新加载或重启你的 Mavis 会话，让新 skill 生效。

## 怎么调用

你可以用类似下面的话触发：

```text
调用现货机器成本核算 skill。
品牌：超微
数量：2 台
采购单价：xx 万
交易方：xxx
图片我发你了，请识别配置并写入 Summary。
```

也可以先给信息，再明确调用：

```text
这批机器品牌是广达，数量 4 台，下面是配置截图。
调用现货机器成本核算 skill，帮我写进表里并拆成本。
```

如果缺少关键信息，agent 应该先问，例如：

```text
品牌会影响价格规则，请补充品牌后我再写入。
```

## 执行流程

skill 的标准执行流程是：

1. **读取表结构**：读取主表和标准价库表头，建立字段映射。
2. **新增并写入基础信息**：默认在主表最底部新增一行，自动延续 item 计数，并按表头语义写入品牌、数量、图片来源、采购价格、交易方、备注等。
3. **识别配置摘要**：把图片或配置文本整理成 `CPU / GPU / 内存 / 系统盘 / 数据盘` 五类。
4. **读取标准价库**：根据部件类型、规格、品牌条件匹配价格。
5. **拆分成本差异**：按 `部件类型 + 规格 + 数量` 对比当前配置和标准配置。
6. **复制公式**：从上一行或最近有效行复制公式并调整到目标行。
7. **自动核对**：检查字段写入、配置摘要、价库命中、拆分字段和公式引用。

## 配置摘要格式

配置摘要固定输出五类：

```text
CPU：...
GPU：...
内存：...
系统盘：...
数据盘：...
```

规则：

- 五类都必须出现。
- 缺失字段写 `待确认`。
- 不从容量反推接口，例如不能只凭 `3.84T` 判断 `E1.S` 或 `U.2`。
- 忽略 `22x110mm` 这类物理尺寸，不能识别成数量。
- 数量只认明确表达，例如 `*2`、`×2`、`x2`、`2块`、`2条` 等。

## 成本拆分逻辑

成本拆分不是按总容量粗算，而是按组件逐项比对。

- 同规格数量不同：只计算数量差。
- 规格不同：标准规格按更换价计入，原规格按拆机售卖价抵扣。
- 品牌专属价格优先，找不到品牌专属规则时再用通用规则。
- 新品牌、新规格、新接口没有明确规则时写 `待确认`。

## 失败处理

| 情况 | 处理方式 |
| --- | --- |
| 图片看不清 | 对应配置写 `待确认` |
| 品牌缺失且影响价格 | 先问用户品牌 |
| 数量缺失 | 先问用户数量 |
| 价库无匹配 | 对应价格或拆分项写 `待确认` |
| 表头无法匹配 | 停止写入，避免错列 |
| 公式来源缺失 | 停止写入公式，等待确认 |
